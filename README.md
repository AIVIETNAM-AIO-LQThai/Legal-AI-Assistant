# Product Handover Documentation

## Project

**Project name:** SME Legal AI Assistant  
**Competition:** Road to AI 2026 - Build AI Legal Assistant  
**Objective:** Build a reproducible Vietnamese legal question-answering system for SME-related legal questions. The system retrieves relevant legal documents and articles, generates Vietnamese legal answers, and formats the final output according to the competition submission requirements.

---

## 1. Data Documentation

### 1.1 Data Sources

The system uses the following data sources:

| Data type | Description | Usage |
|---|---|---|
| Test questions | JSON file provided by the competition, containing `id` and `question` fields | Input questions to be answered |
| Legal documents | Vietnamese legal documents in PDF format | Source corpus for legal retrieval and answer grounding |
| Processed legal data | Extracted text, parsed articles, chunks, metadata, reference graph, SQLite tables, and retrieval indexes | Intermediate artifacts generated by the notebook |
| Submission output | `results.json` | Final prediction file for submission |

If any external data is used, its source, collection method, preprocessing method, and role in the system must be documented clearly.

### 1.2 Input Data Format

The competition test set contains a list of question objects:

```json
[
  {
    "id": 1,
    "question": "Legal question in Vietnamese"
  }
]
```

Rules:

- `id` must be preserved exactly.
- `question` must be preserved exactly and must not be rewritten.

### 1.3 Processed Data Structure

Legal PDFs are processed into a structured retrieval corpus:

```text
Legal PDF
  -> legal document metadata
  -> article records
  -> clause/point-aware chunks
  -> SQLite lookup tables
  -> BM25 index
  -> vector index
  -> legal reference graph
```

Main generated artifacts may include:

```text
data/processed/articles.parquet
 data/processed/chunks.parquet
 data/indexes/legal_pdf.db
 data/indexes/bm25_index.pkl
 data/indexes/chroma/
 data/outputs/results.json
```

The exact paths can be adjusted in the notebook configuration section.

3. Place the question file at the configured input path.
4. Download required checkpoints from the checkpoint link.
5. Place checkpoints in the configured `checkpoints/` folder or update the notebook paths.

Run the notebook from top to bottom in order. The main stages are:

```text

2. Extract and normalize legal PDF text.
3. Parse documents into article-level records.
4. Build clause/point-aware chunks.
5. Build SQLite lookup tables and reference graph.
6. Build BM25 and vector indexes.
7. Load test questions.
8. Retrieve and rerank legal evidence.
9. Select relevant documents and articles.
10. Generate Vietnamese legal answers.
11. Validate submission fields.
12. Export results.json.
```

For a quick check, run the sample/debug cells first. For the final result, run the full submission generation cell.

## 5. Submission Format

### 5.1 `results.json` Structure

The final output must be a JSON list. Each item corresponds to one test question:

```json
[
  {
    "id": 1,
    "question": "",
    "answer": "",
    "relevant_docs": [
      "<document_code>|<document_title>"
    ],
    "relevant_articles": [
      "<document_code>|<document_title>|<article>"
    ]
  }
]
```

Field requirements:

| Field | Requirement |
|---|---|
| `id` | Must match the original question ID |
| `question` | Must keep the original question text unchanged |
| `answer` | Must be a Vietnamese legal answer with clear legal basis |
| `relevant_docs` | List of related legal documents in the required format |
| `relevant_articles` | List of related legal articles in the required format |

Document format:

```text
<document_code>|<document_title>
```

Article format:

```text
<document_code>|<document_title>|<article>
```

Example:

```text
04/2017/QH14|Luật 04/2017/QH14 Luật Hỗ trợ doanh nghiệp nhỏ và vừa|Điều 4
```

### 5.2 ZIP Submission Structure

The submission file must be a `.zip` file containing only `results.json` at the root level.

Valid structure:

```text
submission.zip
└── results.json
```

Invalid structure:

```text
submission.zip
└── folder/
    └── results.json
```

## 7. Validation Before Submission

Before submitting, verify:

- `results.json` is valid JSON.
- The file is a list of prediction objects.
- Every object contains `id`, `question`, `answer`, `relevant_docs`, and `relevant_articles`.
- Every `question` matches the original input question.
- Every answer is in Vietnamese.
- Every relevant document follows `<document_code>|<document_title>`.
- Every relevant article follows `<document_code>|<document_title>|<article>`.
- `results.json` is placed directly at the root of the `.zip` file.
- No folder wraps `results.json` inside the zip.
- All data and checkpoint links are accessible.
- The README can reproduce the system from a clean environment.

## 8. Known Limitations

- The system depends on the quality of PDF extraction and legal text normalization.
- Some legal documents may contain complex formatting, tables, amendments, or cross-references that are difficult to parse perfectly.
- Retrieval quality may vary for questions requiring multiple legal domains or implicit legal reasoning.
- The generated answers are for competition and reference purposes only and should not be treated as official legal advice.

This project is developed for the R2AI 2026 competition and educational research purposes. The generated answers are not official legal advice. Important legal decisions should be verified against official legal documents or reviewed by qualified legal professionals.
